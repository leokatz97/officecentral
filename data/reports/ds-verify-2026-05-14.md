| collection.business-furniture.json | ds-cc-base | STALE-KEY | Template has setting `"cta_url": "/pages/quote"` but ds-cc-base schema does not define a `cta_url` setting id. **Schema drift count: 1**

Fix: Add `cta_url` as a `url` type setting in ds-cc-base schema, or remove the key from collection.business-furniture.json if the CTA URL is hardcoded in the section Liquid.

## Recommendation

### Section cleanup (post-launch, low priority)
- **ORPHANED-DEFINED: 0** — nothing to clean up in the BBI section set.
- **LEGACY-STARLITE: 64 sections** — all candidates for removal post-launch once the Shopify store is fully migrated to BBI templates. Priority removal candidates:
  1. `main-product`, `main-collection`, `main-article`, `main-blog`, `main-search`, `main-404` — direct Starlite replacements for BBI DS sections; confirmed unused.
  2. `header`, `footer`, `cart` — replaced by `bbi-nav-wrap`, `bbi-footer-wrap`, `ds-cart-base`.
  3. `ajax-cart-drawer`, `predictive-search`, `pickup-availability` (no-schema sections) — audit JS dependencies before removing.
  4. Remaining decorative/marketing sections (`countdown`, `parallax`, `video-banner`, etc.) — low risk, safe to bulk-remove.

### Schema drift fix
- **`collection.business-furniture.json`** has a stale `cta_url` key not in ds-cc-base schema. Add the schema setting or clean the template JSON. Renders fine but the value is invisible to the Theme Editor.

### Index homepage (custom-liquid)
- The homepage (`index.json`) uses 8 raw `custom-liquid` sections for the BBI homepage blocks. These work but bypass the BBI design system section pattern. Post-launch consideration: migrate homepage blocks to proper BBI sections for Theme Editor editability.

### CSS tokens
- Token system is clean. `ds-landing.css` is the single source of truth for `--bbi-*` variables. No orphaned or undefined references found.
