# Topic Cluster Templates

Planning worksheets, architecture patterns, and measurement frameworks for building topic clusters.

## Topic Cluster Planning Worksheet

### Step 1: Define the Pillar Topic

| Field | Your Input |
|-------|-----------|
| **Pillar Topic** | [Broad topic name] |
| **Pillar Keyword** | [Head keyword, 1-3 words] |
| **Monthly Search Volume** | [Volume] |
| **Keyword Difficulty** | [KD score] |
| **Business Relevance** | [1-5 scale] |
| **Current Ranking** | [Position or "Not ranking"] |
| **Competitor Coverage** | [How many have pillar content?] |
| **Estimated Cluster Size** | [Number of subtopics] |
| **Content Assets Available** | [Existing content to repurpose/link] |

**Pillar viability checklist:**
- [ ] At least 8-12 subtopics identified
- [ ] Combined cluster keyword volume >5,000/month
- [ ] Topic directly relevant to your product/service
- [ ] You can provide unique expertise or data
- [ ] Competitors have not yet built a dominant cluster

### Step 2: Map Cluster Subtopics

| # | Subtopic | Target Keyword | Volume | KD | Intent | Format | Status |
|---|---------|---------------|--------|-----|--------|--------|--------|
| 1 | [Subtopic] | [Long-tail keyword] | [Vol] | [KD] | [I/N/C/T] | [Guide/Tutorial/List] | [Idea/Draft/Published] |

### Step 3: Define Internal Linking Map

| Source Page | Links To | Anchor Text Strategy |
|------------|---------|---------------------|
| Pillar | Cluster 1, 2, 3... (all) | Descriptive, keyword-relevant |
| Cluster 1 | Pillar + Cluster 2, 3 | Natural contextual links |

**Linking rules:**
- Every cluster page MUST link to the pillar page
- The pillar page MUST link to every cluster page
- Cluster pages SHOULD link to 2-3 related cluster pages
- Use descriptive anchor text (not "click here")
- Place links within body content, not just footer lists

## Hub-and-Spoke Architecture Patterns

### Pattern 1: Classic Hub-Spoke (Educational Topics)

```
                    Pillar:
                    Complete Guide
                    to [Topic]
                   / | | | | \
    What is?  Benefits  Examples  Tools  Mistakes  Checklist
```

### Pattern 2: Layered Cluster (Technical Topics)

```
    Pillar ---- Beginner Cluster (Getting Started, Basics, First Steps)
       |------- Intermediate Cluster (Advanced Techniques, Patterns)
       |------- Advanced Cluster (Expert Strategies, Edge Cases)
```

### Pattern 3: Use-Case Cluster (Product/Service Topics)

```
    Pillar ---- [Topic] for SMBs
       |------- [Topic] for Enterprise
       |------- [Topic] for Agencies
       |------- [Topic] for Ecommerce
```

### Pattern 4: Process Cluster (How-To Topics)

```
    Pillar --> Step 1: Research --> Step 2: Plan --> Step 3: Execute --> Step 4: Measure
                   |                  |                |                  |
               Sub-guide          Template          Tutorial          Dashboard Guide
```

## Internal Linking Within Clusters

### Linking Density Guidelines

| Cluster Size | Min Links/Page | Max Links/Page | Pillar Link Density |
|-------------|---------------|---------------|-------------------|
| 5-8 pages | 2-3 | 5-6 | Link to every cluster page |
| 9-15 pages | 3-4 | 6-8 | Link to every cluster page |
| 16+ pages | 4-5 | 8-10 | Link to top cluster pages, categorize rest |

### Anchor Text Strategy

| Link Type | Approach | Example |
|-----------|---------|---------|
| Cluster -> Pillar | Broad keyword or branded | "our complete guide to keyword research" |
| Pillar -> Cluster | Specific keyword | "learn about long-tail keyword strategies" |
| Cluster -> Cluster | Contextual, conversational | "this connects to how you assess keyword difficulty" |

### Anti-Patterns to Avoid

| Anti-Pattern | Fix |
|-------------|-----|
| Footer-only links to cluster pages | Move links into body content |
| Generic anchors ("click here") | Use descriptive, keyword-relevant anchors |
| Orphan cluster pages | Add links from pillar and related clusters |
| Over-linking (20+ per page) | Keep to 5-10 relevant internal links |
| Linking only to pillar | Link to 2-3 related sibling pages |

## Content Calendar Integration

### Cluster Build Sequence

| Phase | What to Publish |
|-------|----------------|
| Phase 1 | Pillar page first (establishes hub) |
| Phase 2 | 3-4 lowest-difficulty cluster pages (early momentum) |
| Phase 3 | Highest-volume cluster pages (leverage pillar authority) |
| Phase 4 | Remaining niche subtopics (fill gaps) |
| Phase 5 | Refresh pillar with new links; update outdated clusters |

### Publishing Cadence

| Team Size | Cluster Build Time | Cadence |
|-----------|-------------------|---------|
| Solo | 6-8 weeks/cluster | 1 per quarter |
| Small (2-3) | 3-4 weeks/cluster | 1 per month |
| Medium (4-6) | 2-3 weeks/cluster | 2 per month |
| Large (7+) | 1-2 weeks/cluster | 1 per week |

## Cluster Performance Measurement

### Key Metrics

| Metric | Target |
|--------|--------|
| Total cluster traffic | Growing month-over-month |
| Keyword coverage | 50+ keywords per mature cluster |
| Average position | Improving toward top 10 |
| Cluster completeness | 100% within planned timeframe |

### Performance Review Cadence

| Timeframe | What to Review | Action If Underperforming |
|-----------|---------------|--------------------------|
| 2 weeks | Indexing status | Fix crawl errors |
| 1 month | Early rankings, traffic | Optimize titles/metas, add internal links |
| 3 months | Ranking positions, trends | Content refresh, add subtopics, build links |
| 6 months | Full performance | Major update or strategic pivot |

### Cluster Health Scorecard (Quarterly)

| Dimension | 1 (Poor) | 3 (Average) | 5 (Excellent) |
|-----------|----------|-------------|---------------|
| Traffic growth | Declining | Flat | Growing 10%+ MoM |
| Keyword coverage | <20 keywords | 20-50 | 50+ |
| Top 10 rankings | 0 in top 10 | 1-5 | 5+ |
| Content freshness | Not updated 12+ months | Updated within 6 months | Within 3 months |
| Internal linking | Missing links, orphans | Basic linking | Full cross-linking |
| Completeness | <50% subtopics | 50-80% | 80-100% |

**Health Score** = Average of dimensions. Below 3.0 = at risk; below 2.0 = strategic review needed.
